## Membership Philosophy

iTowns welcomes participation from everyone. Membership levels recognize different levels of contribution and commitment, with corresponding privileges and responsibilities.

## Membership Levels

### Community Member

**Who**: Anyone using or interested in iTowns

**Requirements**: None

**Privileges**:
- Participate in public discussions
- Report issues
- Submit feature requests
- Use the framework

**Responsibilities**:
- Follow code of conduct
- Respect community norms

---

### Contributor

**Who**: Regular contributors to code, docs, or community

**Requirements**:
- 3+ merged contributions (code, docs, or substantial issues)
- Active participation for 3+ months
- Nominated by existing contributor

**Privileges**:
- Voting rights on technical decisions
- Listed on contributors page
- Contributor badge
- Access to contributor channels

**Responsibilities**:
- Respond to issues in area of expertise
- Review PRs when requested
- Participate in discussions

---

### Core Maintainer

**Who**: Experienced contributors with sustained commitment

**Requirements**:
- 6+ months as contributor
- Significant contributions across multiple areas
- Demonstrated technical judgment
- Nominated and approved by existing maintainers

**Privileges**:
- Merge access to repositories
- Release management authority
- Technical decision-making
- Attend maintainer meetings

**Responsibilities**:
- Regular code review
- Mentoring contributors
- Release preparation
- Respond to security issues

---

### Governance Board Member

**Who**: Strategic leaders representing community interests

**Requirements**:
- 12+ months as maintainer OR significant organizational contribution
- Demonstrated commitment to project sustainability
- Elected by community vote

**Privileges**:
- Strategic decision-making
- Partnership approval
- Funding allocation
- Governance policy changes

**Responsibilities**:
- Attend monthly board meetings
- Represent community interests
- Long-term planning
- Conflict resolution

---

## Organizational Membership

Organizations can participate at different levels:

### User Organization

**Commitment**: Using iTowns in production

**Benefits**:
- Listed in users directory (optional)
- Support channel access
- Feature request prioritization

---

### Contributing Organization

**Commitment**: Regular contributions (code, docs, or funding)

**Benefits**:
- Logo on website
- Case study opportunity
- Advisory input on roadmap
- Priority support

**Requirements**:
- Minimum 100 hours/year contribution OR
- Financial contribution (sliding scale based on organization size)

---

### Founding Organization

**Commitment**: Foundational support

**Benefits**:
- Governance Board seat
- Major decision input
- Featured partnership status

**Requirements**:
- Significant ongoing contribution
- Multi-year commitment
- Approved by Governance Board

---

## How to Join

### Individuals

1. Start contributing (code, docs, issues)
2. Engage consistently for 3+ months
3. Request contributor status or await nomination
4. Approval by existing contributors

### Organizations

1. Contact governance board
2. Discuss appropriate membership level
3. Agree on contribution terms
4. Formal welcome and onboarding
